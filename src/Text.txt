## react intro

    What is React?

    1) React is a JavaScript library (not a framework) developed by Facebook (Meta) for building user interfaces (UIs), especially Single Page Applications (SPAs).

    2) It helps developers build reusable UI components that update efficiently when data changes.

    Instead of reloading a whole webpage when something changes, React updates only the parts of the UI that need to change.

    -------
    Why do we need React?

	It makes UI development faster, cleaner, and more efficient.

	Provides a better user experience with smooth updates without page reloads.

## useState() & forms

    useState is a React hook that allow you to add state (datathat can change over time)to a
    functional component.

    Normailly, in js function ,variables disappear the function runs, but with useState ,
    react remamber values between re-render.

    syntax:

        const [state,setState]=useState(initialValue)

        state--  the current value of the state variables.
        setState-- a function that update the state.
        initialValue-- the Starting value of the state.

## useEffect()

    In react,useEffect is a hook that lets you perform side effects in functional component.
    Side effect  are task that happen ouside the normal Reacct rendering process ,like:

            Fetching data from an API
            Updating the  Browser DOM manually
            Starting /stopping timers

        syntax:

            useEffect(()=>{
                //code
            },[dependency 1,dependency 2,dependency 3....])

            [] --- if that dependency is empty means that useEffect will be run only once at begining

## Routing in react && nested routing.

    Routing means navigation between different Pages in a web app without refrehing the  browser.

    In traditinal website, clicking a link reloads the  whole page.

    In react (SPA), Routing let us change the view without reloading.

            Better performance
            Smoother user Experiance.
            Control over navigation without server request.

    comment: npm install react-router-dom

    Structure of Routing

        <BrowserRouter>
            <Routes>
                <Route path="Address path" element={<Component/>}/>
                <Route path="/Home" element={<Home/>}/>
                <Route path="/about" element={<Abou/>}/>
                <Route path="/contact" element={<Contact/>}/>
                <Route path="/Service" element={<Service/>}/>
            </Routes>
        </BrowserRouter>


    BrowserRouter

        it is the main wrapper component for routing inn reeact App.
        It enables navigation using the  browser's URL (history url).
        Without this, routing will not work.

    Routes

        It contoins all the <Route/> component
        It tells react router which component to show for which path.
        Only one route will match and render at atime.

    Route 

        Defines a specifi url path and which component should load for that path.
